# 销售订单 PRD

版本：v1.0  
日期：2026-05-11  
来源：`原型解释说明260507.docx`、当前销售订单设计稿、`财务报表PRD_v1.3_2026-05-11.md`、`销售订单修改记录.md`、销售订单字段改动与财务报表验证结果  
范围：订单管理 - 销售订单列表、销售订单详情、新增/编辑、费用分摊、财务核算

## 1. 结论

销售订单不是单纯的订单登记页，而是财务报表、部门报表、运营报表、SKU 报表、市场报表、主体报税口径的基础核算单据。

销售订单金额字段需要区分 `录入口径` 和 `展示口径`：

| 场景 | 口径 |
|---|---|
| 创建/编辑页面 | 支持按业务发生币种录入，可选择 USD、RMB 或对应本地币种；系统按汇率折算生成 RMB 金额 |
| 销售订单列表页 | 作为汇总查看页面，金额字段默认展示 RMB，不重复展示原币金额和币种 |
| 销售订单详情页 | 主展示 RMB；同时保留原币金额、币种、汇率、汇率日期、汇率来源和折算记录 |
| 财务报表取数 | 统一取 RMB 折算金额，原币金额仅用于追溯、核对和重新计算 |

当前设计稿已基本满足销售订单主流程，但开发前必须补齐 4 个约束：

| 优先级 | 必须补齐项 | 原因 |
|---|---|---|
| P0 | `删除` 改为 `作废/取消` | 销售订单是财务核算来源，不能物理删除；否则历史报表和审计断链 |
| P0 | 汇率和原币折算留痕 | 汇率会影响净销售额、销售净额、毛利润；原币、币种、汇率、RMB 折算金额必须保存 |
| P0 | 费用分摊来源和审核状态 | 广告费、仓储费、盘点、清算、其他费用必须能追溯来源 |
| P1 | 公司主体、店铺、运营归属快照 | 店铺主体或运营归属后续可能变化，历史订单必须按订单发生时归属统计 |

## 2. 用户与业务目标

| 用户角色 | 目标 | 关注点 |
|---|---|---|
| 运营 | 快速录入销售订单、退款、平台补偿金、平台费用 | 订单信息准确、录入负担低、可批量导入 |
| 跟单 | 补充运输方式、头程/尾程/国内直发费用、尾程编号和发货日期 | 运费准确、国内直发与头尾程互斥 |
| 财务 | 核对订单收入、费用分摊、汇率、税点、毛利润、主体报税口径 | 可追溯、可审核、可导出 |
| 主管 | 查看部门、店铺、运营、SKU、市场维度经营结果 | 报表准确、归属不重复 |
| 总经理 | 查看公司整体净销售额、成本、费用、毛利润、净利润 | 指标闭环、异常可下钻 |

| 修改汇率 | 是 | 汇率修改需记录历史 |
| 费用分摊明细 | 是 | 支撑广告费、仓储费、盘点、清算、其他费用分摊 |

| 回款登记 | 否 | 回款来自店铺回款记录，不直接放在销售订单主表 |

## 4. 业务流程

```mermaid
flowchart LR
  A["运营新增/导入销售订单"] --> B["系统带出平台、店铺、运营、公司主体、采购价"]
  B --> C["运营录入平台订单号、SKU、数量、净销售额、退款/补偿"]
  C --> D["跟单补充运输方式、头程、尾程、国内直发、尾程编号"]
  D --> E["系统计算净销售额RMB、采购成本、损耗、税点"]
  E --> F["财务录入或审核费用分摊"]
  F --> G["系统计算毛利润、毛利润率"]
  G --> H["进入财务报表、部门报表、运营报表、SKU报表、市场报表"]
```

## 5. 销售订单列表

### 5.1 查询条件

| 查询项 | 类型 | 说明 |
|---|---|---|
| 订单编号 | 输入框 | 支持销售订单编号模糊搜索 |
| 平台订单号 | 输入框 | 支持平台订单号模糊搜索 |
| 运营人员 | 下拉框 | 默认按当前账号权限过滤 |
| 店铺 | 下拉框 | 选择平台后联动店铺 |
| 成交时间 | 日期范围 | 默认近一个月 |
| 平台 | 下拉框 | Amazon、Temu、eBay、Wayfair、Walmart、独立站、国际站等 |
| 状态 | 下拉框 | 正常、取消、作废 |

### 5.2 列表按钮

| 按钮 | 规则 |
|---|---|
| 新增销售订单 | 有新增权限的运营、主管、财务可见 |
| 导入 | 支持历史订单或批量订单导入 |
| 导出 | 按当前筛选条件导出 |
| 批量作废/取消 | 仅主管、财务、总经理可操作；需填写原因 |

### 5.3 列表字段

列表页用于快速查看订单核心信息和财务结果，不承担多币种录入和复杂核算说明。

展示规则：

| 模块 | 规则 |
|---|---|
| 金额展示 | 列表页金额字段统一展示 RMB 折算金额 |
| 币种展示 | 不在每个金额字段后重复展示 USD、EUR、CAD 等原币信息 |
| 汇率信息 | 默认不在列表展示，通过详情页查看 |
| 财务统计 | 列表页展示结果需与财务报表使用同一 RMB 口径 |
| 平台佣金 | 不进入默认列表；用于销售净额和税费报表 |

完整核算过程、原币和汇率追溯在详情页查看。

| 字段 | 说明 |
|---|---|
| 订单编号 | 系统生成，唯一 |
| 平台 | 订单所属平台 |
| 店铺 | 订单所属店铺 |
| 运营人员 | 订单归属运营 |
| 市场 | 美国、欧洲、加拿大、澳大利亚、其他等 |
| 国家 | 买家国家 |
| 区域 | 美国订单展示美东、美中、美西；非美国可为空 |
| 州 | 美国订单州 |
| 成交时间 | 订单成交日期 |
| SKU | 支持一个订单多个 SKU 时按订单行展示 |
| 销量 | 商品数量 |
| 零售价 | 可作为参考字段；如列表空间不足，建议不展示或仅展示 RMB |
| 净销售额RMB | 订单明细净销售额汇总 |
| 采购成本 | 采购价 * 数量汇总 |
| 头程费用 | 订单头程费用汇总 |
| 尾程费用 | 订单尾程费用汇总 |
| 国内直发 | 国内直发费用 |
| 广告费用 | 折算后的 RMB |
| 亚马逊仓储费 | 折算后的 RMB |
| 海外仓仓储费 | 根据入库时间和海外仓模板自动计算后的 RMB |
| 退款金额 | 退款金额 RMB |
| 平台补偿金 | 平台补偿金额 RMB |
| 盘点 | Amazon 相关费用折算后的 RMB |
| 清算 | Amazon 清算收入折算后的 RMB |
| 其他费用 | 公共费用或手工费用折算后的 RMB |
| 损耗 | 系统按比例计算后的 RMB |
| 税点 | 系统按比例计算后的 RMB |
| 毛利润 | 系统计算 |
| 状态 | 正常、取消、作废 |
| 操作 | 查看、编辑、作废/取消 |

### 5.4 不建议在列表展示的字段

| 字段 | 原因 | 建议处理 |
|---|---|---|
| 原币金额 | 会导致每个金额字段重复展示，列表过宽 | 放到订单详情页 |
| 原币币种 | 列表页不承担核算追溯 | 放到订单详情页 |
| 汇率 | 列表页不是折算核对页面 | 放到详情页或操作日志 |
| 汇率来源 | 对列表查询效率和阅读效率帮助有限 | 放到详情页或操作日志 |
| 平台佣金 | 可在订单创建/编辑中维护，但不需要在订单总表默认展示 | 用于销售净额和税费报表 |

### 5.5 操作规则

| 操作 | 规则 |
|---|---|
| 查看 | 所有有权限用户可查看 |
| 编辑 | 按字段权限控制 |
| 作废/取消 | 替代删除；需填写原因；作废后默认不进入财务报表 |
| 删除 | 不提供物理删除 |

## 6. 销售订单详情

详情页按以下区块组织：

1. 基础信息
2. 运输信息
3. 商品信息
4. 财务汇总信息
5. 原币折算明细
6. 费用来源明细
7. 分摊明细表
8. 财务计算结果
9. 操作记录
10. 系统信息

详情页展示规则：

| 模块 | 规则 |
|---|---|
| 主金额展示 | 订单详情页主展示金额统一为 RMB |
| 原币追溯 | 保留原币金额、原币币种、汇率、折算后 RMB |
| 汇率信息 | 展示汇率值、汇率日期、汇率来源 |
| 数据来源 | 区分系统同步、平台账单、物流账单、仓储账单、人工录入、财务调整、后台配置 |
| 修改记录 | 人工修改金额、币种、汇率时记录修改人、修改时间、修改前后值 |

### 8.1 国内直发互斥规则

```text
如果 运输方式 = 国内直发：
  国内直发RMB > 0
  头程运费RMB = 0
  尾程运费RMB = 0
  毛利润公式中使用 国内直发RMB

如果 运输方式 != 国内直发：
  国内直发RMB = 0
  按实际情况计算头程运费RMB、尾程运费RMB
```

## 9. 商品信息

销售订单需支持一张订单多条商品明细。财务统计 SKU、品类、市场、运营利润时，优先按订单明细行拆分。

| 字段 | 类型 | 来源/规则 | 是否必填 | 责任角色 |
|---|---|---|---|---|
| 产品编号/SKU | 下拉框 | 从产品基础数据选择 | 是 | 运营 |
| 数量 | 数字 | 销售数量 | 是 | 运营 |
| 零售价原币金额 | 金额 | 平台零售价；可按业务发生币种录入 | 否 | 运营 |
| 零售价币种 | 枚举 | USD、RMB 或对应本地币种 | 否 | 运营 |
| 净销售额原币金额 | 金额 | 扣除促销、折扣、优惠券、佣金后的到手价格 | 是 | 运营 |
| 净销售额币种 | 枚举 | USD、RMB 或对应本地币种 | 是 | 运营 |
| 平台佣金原币金额 | 金额 | 运营录入，用于销售净额计算 | 是 | 运营 |
| 平台佣金币种 | 枚举 | USD、RMB 或对应本地币种 | 是 | 运营 |
| 汇率 | 数字 | 默认取基础数据汇率，可修改 | 是 | 系统/财务 |
| 汇率日期 | 日期 | 使用哪一天的汇率 | 是 | 系统 |
| 汇率来源 | 枚举 | 系统汇率表、平台账单、人工维护等 | 是 | 系统 |
| 手续费率 | 数字 | 默认 0.9975，可由基础数据配置 | 是 | 系统 |
| 净销售额RMB | 金额 | 净销售额原币金额 * 汇率 * 手续费率；若原币为 RMB 则直接使用 | 是 | 系统 |
| 平台佣金RMB | 金额 | 平台佣金原币金额 * 汇率；若原币为 RMB 则直接使用 | 是 | 系统 |
| 采购价RMB | 金额 | 从产品基础数据带出 | 是 | 系统 |
| 采购成本RMB | 金额 | 采购价RMB * 数量 | 是 | 系统 |
| 头程分摊RMB | 金额 | 按订单/SKU/批次分摊 | 否 | 系统 |
| 尾程分摊RMB | 金额 | 按订单或尾程账单分摊 | 否 | 系统 |
| 国内直发分摊RMB | 金额 | 国内直发订单使用 | 否 | 系统 |
| 备注 | 文本 | 商品行备注 | 否 | 运营 |

### 9.1 汇率修改规则

| 规则 | 说明 |
|---|---|
| 默认汇率 | 新增订单时取基础数据当前美元汇率 |
| 修改权限 | 财务、主管、总经理可修改；运营是否可改由权限配置 |
| 留痕字段 | 修改前汇率、修改后汇率、修改原因、修改人、修改时间 |
| 历史订单 | 历史订单按订单保存的历史汇率计算，不随基础汇率变化 |

### 9.2 多币种金额存储规则

每个涉及多币种的金额字段，至少保存以下数据：

| 字段 | 说明 |
|---|---|
| 原币金额 | 用户录入、平台同步或账单导入的原始金额 |
| 原币币种 | USD、RMB、EUR、CAD 等 |
| 汇率 | 原币折算 RMB 的汇率 |
| 汇率日期 | 使用哪一天的汇率 |
| 汇率来源 | 系统汇率表、平台账单、人工维护等 |
| RMB金额 | 财务报表、销售订单列表使用的统一金额 |
| 数据来源 | 平台同步、人工录入、账单导入、系统计算等 |

### 9.3 创建/编辑页面录入规则

创建/编辑页面用于录入订单原始业务数据，需要支持按实际发生币种录入，并自动生成统一 RMB 口径。

| 字段 | 创建/编辑录入方式 | 系统处理规则 |
|---|---|---|
| 零售价 | 可录入原币金额 | 作为参考字段，不作为核心财务统计依据 |
| 净销售额 | 支持选择币种录入，如 USD、RMB 或对应本地币种 | 系统根据汇率生成 RMB 金额；财务报表取 RMB |
| 平台佣金 | 运营录入，支持选择币种 | 系统折算为 RMB，用于销售净额 |
| 采购成本 | RMB | 采购价默认 RMB，不需要多币种 |
| 头程运费 | RMB | 头程默认 RMB |
| 尾程运费 | 默认支持 USD，也允许 RMB 或对应本地币种 | 系统统一折算为 RMB；列表和详情主展示 RMB |
| 国内直发 | RMB | 国内直发费用默认 RMB |
| 广告费用 | 支持平台原币或 RMB | 平台后台若为 USD 等原币，系统折算为 RMB；如果已经是 RMB，则直接使用 |

## 12. 财务计算结果

| 字段 | 计算/来源 | 说明 |
|---|---|---|
| 退款金额 | 运营录入或平台数据导入 | 统一折算 RMB |
| 平台补偿金 | 运营录入或平台数据导入 | 平台端或买家端原因补偿给公司的金额 |
| 售后金额 | 退款金额 - 平台补偿金 | 财务报表使用 |
| 广告费用 | 分摊明细汇总 | 成本项 |
| 亚马逊仓储费 | 分摊明细汇总 | 成本项 |
| 海外仓仓储费 | 根据入库时间和海外仓费用模板自动计算，结果写入费用明细 | 成本项 |
| 盘点 | 分摊明细汇总 | 成本项 |
| 清算 | 分摊明细汇总 | 收入加项 |
| 其他费用 | 分摊明细汇总 | 成本项 |
| 损耗 | 净销售额RMB * 损耗比例 | 比例从后台配置，2026 年默认 3.5% |
| 税点 | 净销售额RMB * 税点比例 | 比例从后台配置，2026 年默认 6% |
| 毛利润 | 系统计算 | 见公式 |
| 毛利润率 | 毛利润 / 净销售额RMB | 净销售额为 0 时展示 `--` |

### 12.1 毛利润公式

```text
毛利润 =
净销售额RMB
- 采购成本RMB
- 头程运费RMB
- 尾程运费RMB
- 国内直发RMB
- 广告费用
- 亚马逊仓储费
- 海外仓仓储费
- 退款金额
+ 平台补偿金
- 盘点
+ 清算
- 其他费用
- 损耗
- 税点
```

### 12.2 毛利润率公式

```text
毛利润率 = 毛利润 / 净销售额RMB
```

### 12.3 售后金额公式

```text
售后金额 = 退款金额 - 平台补偿金
```

### 12.4 销售净额公式

```text
销售净额 = 净销售额RMB + 平台佣金RMB - 退款金额RMB
```

说明：

- 经营分析使用 `净销售额RMB`。
- 主体报税使用 `销售净额`。
- `平台佣金` 由运营录入；即使不在列表主展示，也必须作为可取数字段保存。

| 操作人 | 系统生成 | 系统当前用户 |
| 操作时间 | 系统生成 | 操作发生时间 |

## 16. 校验规则

| 场景 | 校验规则 |
|---|---|
| 新增订单 | 成交时间、平台、店铺、运营、平台订单号、产品编号、数量、净销售额必填 |

## 17. 状态规则

| 状态 | 含义 | 是否进入财务报表 |
|---|---|---|
| 正常 | 有效销售订单 | 是 |
| 取消 | 订单取消，未形成有效销售 | 否 |
| 作废 | 录入错误或业务确认无效 | 否 |

状态变更规则：

```text
正常 -> 取消：允许，需原因
正常 -> 作废：允许，需原因
取消 -> 正常：仅财务/主管/总经理可恢复，需原因
作废 -> 正常：仅财务/主管/总经理可恢复，需原因
```

## 22. 已确认口径与待确认问题

### 22.1 已确认口径

| 口径 | 结论 |
|---|---|
| 销售净额 | 销售净额 = 净销售额 + 平台佣金 - 退款金额 |
| 平台佣金 | 平台佣金由运营录入 |
| 海外仓仓储费 | 根据入库时间和海外仓费用模板自动计算仓储费 |
| 回款 | 回款只到店铺维度 |

### 22.2 待确认问题

| 问题 | 影响 |
|---|---|
| 广告费默认按净销售额占比分摊是否所有平台适用 | 影响 SKU、运营、部门利润准确性 |
| 订单取消后是否允许恢复 | 影响状态流转和审计 |

## 23. 验收标准

| 验收项 | 标准 |
|---|---|
| 订单新增 | 能新增单 SKU 和多 SKU 销售订单 |
| 创建/编辑多币种录入 | 净销售额、尾程、广告费、仓储费、退款、平台补偿金等字段可按配置选择币种 |
| 自动折算 RMB | 选择非 RMB 币种后，系统可根据汇率生成 RMB 金额 |
| 列表展示一致 | 销售订单列表页金额字段统一展示 RMB，不重复展示原币金额和币种 |
| 详情可追溯 | 销售订单详情页可以查看原币金额、币种、汇率、汇率来源、折算后 RMB |
| 角色必填 | 运营、跟单、财务只校验各自负责字段 |
| 国内直发 | 国内直发与头程/尾程互斥生效 |
| 汇率 | 修改汇率后保留历史记录，历史订单不随基础汇率变化 |
| 分摊 | 广告费、仓储费、盘点、清算、其他费用可追溯来源和审核状态 |
| 毛利润 | 按公式正确计算 |
| 售后金额 | 按 `退款金额 - 平台补偿金` 计算 |
| 销售净额 | 可计算 `净销售额 + 平台佣金 - 退款金额` |
| 状态 | 作废/取消订单不进入默认财务报表 |
| 报表取数 | 公司、部门、运营、SKU、市场报表可从销售订单取数 |
