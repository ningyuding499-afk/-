# 销售订单字段改动与财务报表验证

## 1. 结论

当前销售订单设计稿已覆盖 `基础信息`、`运输信息`、`商品核心成本` 主链路，但还不能完整支撑财务报表 PRD 的统计口径。

必须补齐以下字段来源或分摊明细：

- `销量/数量`
- `平台佣金`
- `退款金额`
- `平台补偿金`
- `广告费`
- `亚马逊仓储费`
- `海外仓仓储费`
- `盘点`
- `清算`
- `其他费用`
- `损耗`
- `税点`
- `毛利润`
- `毛利润率`

否则公司财务、部门财务、运营财务、SKU、市场、主体报税口径都会存在统计断点。

## 2. 销售订单字段改动清单

| 字段组 | 原型说明要求 | 当前设计稿判断 | 财务报表验证 |
|---|---|---|---|
| 订单基础字段 | `成交时间`、`订单编号`、`平台`、`店铺`、`运营人员`、`平台订单号` | 详情页已体现；列表页也有基础查询和订单列表 | 可支撑日期、平台、店铺、运营筛选 |
| 序号 | 原 Word 提到按月排序序号；后续会议纪要要求删除序号 | 当前详情页未见序号，符合后续方向 | 不建议作为财务统计字段，只保留订单编号唯一性 |
| 国家/区域/邮编/州 | `国家`；美国订单增加 `区域`、`邮编`；设计稿还展示 `州` | 已体现国家、区域、州、邮编 | 可支撑市场、美国区域分析 |
| 运输方式 | `FBA`、`海14`、`海15`、`海16`、`海16加`、`海20`、`海21`、`国内直发`；国内直发时头程、尾程为 0 | 详情页有 `货运方式`、尾程编号、收货地址、日期 | 可支撑头程、尾程、国内直发统计，但需做互斥规则 |
| 商品明细 | `产品编号`、`零售价USD`、`净销售额USD`、`净销售额RMB`、`采购价RMB` | 商品信息已体现产品编号、零售价、净销售额、净销售额RMB、采购价 | 缺 `数量/销量`，财务 PRD 明确销量来自销售订单 |
| 汇率 | `净销售额RMB = 净销售额USD * 美元汇率 * 0.9975` | 有 `修改汇率` 按钮 | 必须保存订单历史汇率、手续费率，不能只取当前基础汇率 |
| 物流费用 | `头程运费RMB`、`尾程运费USD/RMB`、`国内直发RMB` | 商品信息有头程费用、尾程费用RMB、国内直发RMB | 可支撑财报成本项；但尾程 USD 原币未在设计稿展示 |
| 月度分摊费用 | 广告费、亚马逊仓储费、海外仓仓储费、盘点、清算、其他费用按净销售额占比分摊 | 当前只有 `分担明细表`，字段过粗 | 分摊明细需增加费用类型、费用月份、币种、汇率、来源单、审核状态 |
| 售后相关 | 退款金额、平台补偿金 | 当前详情截图未见 | 财报的售后金额 = 退款金额 - 平台补偿金，必须补 |
| 自动计算项 | 损耗 3.5%、税点 6%、毛利润、毛利润率 | 当前详情截图未见 | 财报毛利润公式依赖这些字段，必须有计算结果或计算明细 |
| 报税口径 | 销售净额 = 净销售额 + 平台佣金 - 退款金额 | 当前设计未见平台佣金 | 主体报税页无法验证，必须补平台佣金来源 |
| 状态 | 后续纪要要求取消/作废状态，未发货取消不计真实订单 | 当前系统信息仅见 `正常` | 财务统计必须过滤作废/取消订单 |

## 3. 当前设计稿需要调整的点

### 3.1 商品信息表

建议增加以下字段：

| 字段 | 用途 |
|---|---|
| 数量/销量 | 支撑销量、采购成本、SKU、市场、新品、季节性统计 |
| 币种 | 支撑多币种录入和统一 RMB 展示 |
| 订单历史汇率 | 防止后续汇率变化影响历史订单利润 |
| 手续费率 | 支撑 `0.9975` 等手续费口径固化 |
| 净销售额USD原值 | 保留运营录入口径 |
| 净销售额RMB计算值 | 财务报表统一取数口径 |

### 3.2 分担明细表

当前 `分担明细表` 只展示 `修改时间`、`分摊方式`、`费用总额`、`分摊金额`、`产品编号`、`备注`，不足以支撑财务核算。

建议扩展为：

| 字段 | 说明 |
|---|---|
| 费用类型 | 广告费、亚马逊仓储费、海外仓仓储费、盘点、清算、其他费用 |
| 费用月份 | 用于月度费用归集 |
| 原币金额 | 保留原始录入金额 |
| 币种 | USD、RMB 或其他币种 |
| 汇率 | 使用费用发生时汇率 |
| RMB金额 | 财务报表统一统计金额 |
| 分摊规则 | 按净销售额占比、按数量、按重量、按订单金额等 |
| 分摊状态 | 待分摊、已分摊、已审核、已作废 |
| 来源单据 | 关联广告账单、平台账单、财务费用登记、物流账单 |
| 审核人 | 财务审核留痕 |

### 3.3 财务费用信息区块

建议在销售订单详情中新增 `财务费用信息` 区块，集中展示：

- 退款金额
- 平台补偿金
- 广告费
- 亚马逊仓储费
- 海外仓仓储费
- 盘点
- 清算
- 其他费用
- 损耗
- 税点
- 毛利润
- 毛利润率

## 4. 财务报表组合验证

| 财务统计字段 | 销售订单必须提供 | 当前是否足够 | 判断 |
|---|---|---|---|
| 净销售额 | 净销售额USD、汇率、净销售额RMB | 基本够 | 需锁定历史汇率 |
| 销量 | 商品数量 | 不够 | 当前截图未见数量 |
| 采购成本 | 产品编号、采购价、数量 | 不够 | 缺数量，无法计算总采购成本 |
| 头程运费 | 运输方式、头程费用、分摊规则 | 基本够 | 需明确按订单/SKU/批次分摊 |
| 尾程运费 | 尾程费用USD/RMB、尾程账单 | 部分够 | 设计稿只见 RMB，建议保留原币 |
| 国内直发 | 国内直发费用、运输方式互斥规则 | 基本够 | 需保证国内直发时头程/尾程为 0 |
| 广告费用 | 月度广告费、分摊规则、SKU/订单归属 | 不够 | 当前未见完整费用类型 |
| 亚马逊仓储费 | 平台费用登记或分摊结果 | 不够 | 需来源单和分摊状态 |
| 海外仓仓储费 | 海外仓费用登记或系统测算结果 | 不够 | 需按订单/SKU/库存批次归集 |
| 退款金额 | 订单退款金额 | 不够 | 当前未见 |
| 平台补偿金 | 平台补偿金额 | 不够 | 当前未见 |
| 售后金额 | 退款金额 - 平台补偿金 | 不够 | 依赖两个缺失字段 |
| 盘点 | 平台费用登记、分摊结果 | 不够 | 亚马逊专项字段需补 |
| 清算 | 平台费用登记、分摊结果 | 不够 | 亚马逊专项字段需补 |
| 其他费用 | 财务费用登记、分摊结果 | 不够 | 当前分担明细字段不足 |
| 损耗 | 后台比例或订单归集金额 | 不够 | 当前未见 |
| 税点 | 后台税点配置或税费结果 | 不够 | 当前未见 |
| 毛利润 | 完整收入、成本、费用、损耗、税点 | 不够 | 依赖多个缺失字段 |
| 毛利润率 | 毛利润 / 净销售额 | 不够 | 依赖毛利润 |
| 主体报税销售净额 | 净销售额、平台佣金、退款金额、公司主体 | 不够 | 缺平台佣金；公司主体需从店铺带出或保存快照 |
| 部门/运营/SKU/市场报表 | 部门、运营、店铺、平台、SKU、市场、订单明细 | 部分够 | 多 SKU 订单需按明细拆分统计 |

## 6. 关键规则

### 6.1 国内直发互斥规则

```text
如果 运输方式 = 国内直发：
  国内直发费用 > 0
  头程运费 = 0
  尾程运费 = 0
  毛利润公式中使用 国内直发费用

如果 运输方式 != 国内直发：
  国内直发费用 = 0
  按实际情况计算头程运费、尾程运费
```

### 6.2 毛利润公式

```text
毛利润 =
净销售额
- 采购成本
- 头程运费
- 尾程运费
- 国内直发
- 广告费用
- 亚马逊仓储费
- 海外仓仓储费
- 退款金额
+ 平台补偿金
- 盘点
+ 清算
- 其他费用
- 损耗
- 税点
```

### 6.3 毛利润率公式

```text
毛利润率 = 毛利润 / 净销售额
```

### 6.4 售后金额公式

```text
售后金额 = 退款金额 - 平台补偿金
```

### 6.5 主体报税销售净额公式

```text
主体报税销售净额 = 净销售额 + 平台佣金 - 退款金额
```

## 7. 需要重点确认的问题

| 问题 | 影响 |
|---|---|
| `净销售额` 是否已经扣除平台佣金 | 影响主体报税销售净额公式 |
| `平台佣金` 从哪里取数 | 影响报税页和税费管理页 |
| 销售订单是否支持一个订单多 SKU | 影响 SKU、运营、市场维度拆分 |
| 回款是否只到店铺维度 | 影响运营维度回款是否能精确统计 |
| 海外仓仓储费是否自动测算 | 影响是否还需要人工分摊 |
| 广告费按什么粒度分摊 | 影响 SKU、运营、部门利润 |
| 订单取消/作废如何处理 | 影响历史追溯和财务统计准确性 |

## 8. 最终建议

销售订单不要只作为“订单登记表”，应作为财务报表的基础核算单据。

列表页只展示关键经营字段，避免过宽：

- 订单编号
- 成交时间
- 平台
- 店铺
- 运营
- 国家/区域
- 产品编号/SKU
- 数量
- 净销售额RMB
- 采购成本
- 运费合计
- 售后金额
- 毛利润
- 状态

完整收入、成本、费用、分摊、计算过程放到详情页和导出中。

这样才能同时满足：

- 运营快速录单
- 跟单补运输费用
- 财务核对费用和分摊
- 管理层查看公司/部门/运营/SKU/市场利润
- 主体报税销售净额校验
